
`.java` 文件是文本文件，而JVM需要二进制文件，需要将`.java` 转化为`.class`文件
## 1. 类加载区
如果JVM想要执行`.class`文件，就需要将其装进一个**类加载器**，会将所有的`.class`文件全部搬进JVM中
## 2. 方法区
**方法区** 是用于存放类似于元数据信息方面的数据的，比如类信息，常量，静态变量，编译后代码···等
类加载器将 .class 文件搬过来就是先丢到这一块上
## 3. 堆
**堆** 主要放了一些存储的数据，比如对象实例，数组···等，它和方法区都同属于 **线程共享区域** 。也就是说它们都是 **线程不安全** 的
## 4. 栈
**栈** 这是我们的代码运行空间。我们编写的每一个方法都会放到 **栈** 里面运行。**线程独享**
## 5. 程序计数器
主要就是完成一个加载工作，类似于一个指针一样的，指向下一行我们需要执行的代码。
和栈一样，都是 **线程独享** 的，就是说每一个线程都会有自己对应的一块区域而不会存在并发和多线程的问题。

## 创建对象的流程分析
```java
class Person{
	int age = 90;
	String name;
	Person(int a,String n){
		name = n;
		age = a;
	}
}

Person p = new Person(22,"小倩");
```
1. 加载Person类(Person.class),只加载一次
2. `new Person()` 在堆中，开辟一段空间(地址0x1122)，存放age,存放Name;
	- 进行默认初始化
		- `age = 0;
		- `name = null;`
	- 默认初始化后，进行显示初始化
		- `age = 90`
		- `name = null`
	- 构造器初始化：
		- 将**常量池**中的构造的**小倩（0X1133）**，引入**name**，
3. 堆中的`age` 改为20
4. 将堆中的`0X1122`返回到栈中
